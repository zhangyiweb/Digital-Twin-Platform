# 数字孪生平台 - 项目文档

## 📋 项目概述

### 项目简介
数字孪生平台是一个面向设计师的Web端3D场景编辑工具,基于Three.js构建。支持模型导入、材质编辑、场景调试、动画控制等核心功能,最终可打包为静态文件部署到Nginx服务器,或导出独立的可运行代码包。

### 核心特性
- 🎨 **可视化编辑**: 所见即所得的3D场景编辑体验
- 📦 **模型导入**: 支持GLB/GLTF等主流3D格式
- 🔧 **材质编辑**: PBR材质实时调节,支持贴图
- 💡 **灯光控制**: 多种光源类型,实时预览效果
- 🎬 **动画系统**: 播放、剪辑、混合动画
- 🚀 **一键发布**: 导出独立HTML或完整项目代码
- 📊 **性能优化**: 内置性能监控和优化工具

### 技术栈
- **前端框架**: React 18 + TypeScript
- **3D引擎**: Three.js
- **状态管理**: Zustand
- **UI组件**: Ant Design
- **构建工具**: Vite
- **样式方案**: TailwindCSS + CSS Modules
- **部署方式**: 静态文件 (dist/) → Nginx

## 🎯 功能模块详解

### 1. 3D视口系统

#### 功能说明
核心的3D渲染区域,提供模型查看和编辑的可视化界面。

#### 主要功能
- **渲染引擎**: Three.js WebGL渲染器
- **相机控制**: 
  - OrbitControls (轨道旋转/平移/缩放)
  - 透视/正交相机切换
  - 相机预设视角保存
- **交互工具**:
  - TransformControls (移动/旋转/缩放Gizmo)
  - 对象选择 (点击/框选)
  - 选中高亮显示
- **辅助显示**:
  - 地面网格 (GridHelper)
  - 坐标轴 (AxesHelper)
  - 光照可视化

#### 快捷键
| 快捷键 | 功能 |
|--------|------|
| W | 切换移动工具 |
| E | 切换旋转工具 |
| R | 切换缩放工具 |
| F | 聚焦选中对象 |
| Delete | 删除选中对象 |
| Ctrl+Z | 撤销 |
| Ctrl+Y | 重做 |

### 2. 模型导入系统

#### 支持格式
- **GLB** (推荐) - 单文件,包含所有资源
- **GLTF** - JSON格式,配套BIN和贴图
- **OBJ** - 基础3D格式
- **FBX** - 需转换处理

#### 导入方式
1. **拖拽导入**: 直接拖拽文件到视口
2. **文件选择**: 通过文件选择对话框导入
3. **批量导入**: 支持多个模型同时导入

#### 处理流程
```
用户选择文件
    ↓
验证文件格式
    ↓
GLTFLoader解析
    ↓
提取场景信息(网格/材质/动画)
    ↓
自动居中并适配相机
    ↓
添加到场景树
    ↓
生成缩略图
```

### 3. 场景树面板

#### 功能说明
以树形结构展示场景中的所有对象,支持层级管理。

#### 主要功能
- **层级展示**: 显示场景图(Scene Graph)结构
- **节点操作**:
  - 展开/折叠
  - 选择节点
  - 显示/隐藏 (眼睛图标)
  - 锁定/解锁
- **搜索过滤**: 按名称快速查找对象
- **拖拽排序**: 调整层级关系

#### 界面示例
```
📁 Scene
  └─ 📦 Building_01
     ├─ 🔷 Wall_Front
     ├─ 🔷 Wall_Back
     ├─ 🔷 Roof
     └─ 📦 Interior
        ├─ 🔷 Floor
        └─ 🔷 Ceiling
```

### 4. 属性面板

#### 变换属性
- **位置 (Position)**: X, Y, Z 坐标
- **旋转 (Rotation)**: X, Y, Z 欧拉角
- **缩放 (Scale)**: X, Y, Z 缩放因子

#### 材质属性 (PBR)
| 参数 | 范围 | 说明 |
|------|------|------|
| 基础颜色 (Color) | 颜色选择器 | 物体基础颜色 |
| 金属度 (Metalness) | 0-1 | 0=非金属, 1=金属 |
| 粗糙度 (Roughness) | 0-1 | 0=光滑, 1=粗糙 |
| 透明度 (Opacity) | 0-1 | 0=透明, 1=不透明 |
| 自发光 (Emissive) | 颜色+强度 | 自发光效果 |
| 法线贴图 (Normal Map) | 贴图文件 | 表面细节 |

#### 实时预览
所有参数修改即时生效,无需刷新。

### 5. 材质编辑器与替换系统

#### 材质编辑模式 (两种)
**模式一: 参数调整**
- 在原有材质基础上修改参数
- 适合微调颜色、金属度、粗糙度等
- 保留原有材质结构和贴图

**模式二: 材质替换** ⭐
- 完全替换对象使用的材质类型和预设
- 适合快速改变物体外观 (如金属→玻璃→木材)
- 可选择不同材质类型重新配置

#### 材质替换工作流 ⭐

**使用场景**:
1. 快速替换模型材质 (如查看不同材质效果)
2. 批量替换多个对象的材质
3. 从材质库中选择预设应用到模型
4. 自定义材质并保存为预设

**替换流程**:
```
选中对象/多个对象
    ↓
打开材质编辑器 → 切换到"替换"模式
    ↓
选择替换方式:
  ├─ 从预设库选择 (推荐)
  ├─ 从材质库选择 (自定义材质)
  ├─ 从其他对象复制材质
  └─ 创建新材质
    ↓
预览效果 (实时)
    ↓
确认应用 → 替换完成 ✅
```

**替换选项**:
| 选项 | 说明 | 适用场景 |
|------|------|----------|
| **替换全部材质** | 对象所有子网格使用同一材质 | 简单模型快速替换 |
| **按材质槽替换** | 只替换特定材质槽 | 复杂模型部分替换 |
| **保留贴图** | 替换材质类型但保留原贴图映射 | 贴图质量高时 |
| **重置所有参数** | 完全使用新材质预设 | 彻底更换 |
| **混合材质** | 在新旧材质间过渡 | 渐进式调整 |

#### 材质库管理 ⭐

**个人材质库**:
- 创建并保存自定义材质
- 给材质命名和添加标签
- 材质缩略图预览
- 搜索和分类浏览
- 导入/导出材质库 (.dmat格式)

**共享材质库**:
- 团队共享材质资源 (如需要后端)
- 在线材质市场 (可选)
- 导入第三方材质包 (Substance、Quixel等)
- 社区分享材质预设
**材质类型切换** ⭐:
支持在不同Three.js材质类型间切换:

| 材质类型 | 特点 | 适用场景 |
|----------|------|----------|
| **MeshStandardMaterial** | PBR标准,推荐 | 大多数情况 (默认) |
| **MeshPhysicalMaterial** | 物理材质,更真实 | 需要清漆、透光等高级效果 |
| **MeshPhongMaterial** | 经典材质,性能好 | 老旧项目兼容 |
| **MeshLambertMaterial** | 简单漫反射 | 低性能设备 |
| **MeshBasicMaterial** | 不受光照影响 | UI元素、线框 |
| **MeshToonMaterial** | 卡通渲染 | 风格化场景 |
| **ShaderMaterial** | 自定义着色器 | 特殊效果 |

#### 材质预设库 (支持快速替换)
提供常用材质模板,一键应用:
- **金属类**: 铁、铜、铝、金、银、铬、钛合金
- **玻璃类**: 透明玻璃、磨砂玻璃、有色玻璃、防弹玻璃、镜面
- **木材类**: 橡木、胡桃木、松木、红木、竹材、胶合板
- **石材类**: 大理石、花岗岩、石灰石、砂岩、混凝土、板岩
- **塑料类**: 哑光塑料、高光塑料、透明塑料、橡胶、硅胶
- **涂料类**: 乳胶漆、金属漆、荧光漆、清漆、底漆

### 6. 灯光系统

#### 光源类型
1. **环境光 (AmbientLight)**
   - 全局均匀照明
   - 参数: 颜色、强度

2. **平行光 (DirectionalLight)**
   - 模拟太阳光
   - 参数: 颜色、强度、方向、阴影

3. **点光源 (PointLight)**
   - 从一点向四周发光
   - 参数: 颜色、强度、距离、衰减

4. **聚光灯 (SpotLight)**
   - 锥形光束
   - 参数: 颜色、强度、角度、距离、衰减

#### 灯光控制
- 可视化显示 (图标辅助)
- 可选中编辑位置
- 实时预览效果
- 阴影开关和质量控制

---

### 7. 动画系统

#### 功能特性
- **播放控制**: 播放/暂停/停止/循环
- **时间轴**: 可视化动画进度
- **速度控制**: 0.1x - 3x 播放速度
- **混合过渡**: 多个动画平滑切换
- **片段管理**: 查看和选择动画片段

#### 支持的动画类型
- 骨骼动画 (Skeletal Animation)
- 变形动画 (Morph Targets)
- 自定义关键帧动画

### 11. 性能优化系统

   - 按需加载

3. **渲染优化**
   - 实例化渲染 (相同物体)

   - 视锥体剔除
   - 遮挡剔除

### Nginx部署

#### 1. 基础配置
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    root /var/www/digital-twin;
    index index.html;
    
    # SPA路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|glb|gltf)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    # Gzip压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    gzip_min_length 1000;
}
```

#### 2. 部署步骤
```bash
# 1. 将dist目录上传到服务器
scp -r dist/* user@server:/var/www/digital-twin/

# 2. 或使用rsync
rsync -avz dist/ user@server:/var/www/digital-twin/

# 3. 重启Nginx (如需要)
sudo systemctl reload nginx
```

#### 3. 完整Nginx配置示例
```
server {
    listen 80;
    server_name digital-twin.example.com;
    
    # 重定向到HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name digital-twin.example.com;
    
    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;
    
    root /var/www/digital-twin;
    index index.html;
    
    # 主应用
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API代理 (如需要)
    location /api/ {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
    }
    
    # 静态资源优化
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    # 3D模型文件
    location ~* \.(glb|gltf|bin|mtl|obj|fbx)$ {
        expires 30d;
        add_header Cache-Control "public";
        add_header Access-Control-Allow-Origin *;
    }
    
    # Gzip压缩
    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types 
        text/plain 
        text/css 
        application/json 
        application/javascript 
        application/xml 
        image/svg+xml;
    
    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
}
```

## 📖 使用教程

### 快速入门

#### 1. 创建新项目
1. 打开平台,点击"新建项目"
2. 设置场景参数 (背景色、单位等)
3. 进入编辑界面

#### 2. 导入模型
**方式一: 拖拽导入**
- 直接从文件夹拖拽GLB文件到3D视口

**方式二: 文件选择**
- 点击顶部菜单 "文件" → "导入模型"
- 选择模型文件
- 等待加载完成

#### 3. 编辑场景
**选择对象**
- 单击视口中的对象进行选择
- 或在场景树面板中点击节点名称

**调整位置**
- 按 W 键切换移动工具
- 拖动Gizmo控制柄调整位置
- 或在属性面板输入精确数值

**修改材质**
- 选中对象后,右侧显示属性面板
- 展开"材质"部分
- **参数调整模式**: 调整颜色、金属度、粗糙度等参数,实时预览效果
- **材质替换模式**: 点击"替换材质"按钮,从预设库中选择新材质,一键应用

#### 4. 添加灯光
1. 点击顶部工具栏 "添加灯光"
2. 选择灯光类型 (平行光/点光源/聚光灯)
3. 在场景中调整灯光位置
4. 在属性面板调节参数

#### 5. 保存项目
- 自动保存: 系统定时保存
- 手动保存: Ctrl + S
- 导出项目: "文件" → "导出项目" 下载.dtwin文件

#### 6. 发布场景
**导出独立HTML**
1. 点击 "发布" → "导出HTML"
2. 选择导出选项
3. 下载HTML文件
4. 双击HTML文件即可在浏览器查看

**导出项目代码**
1. 点击 "发布" → "导出代码包"
2. 下载ZIP压缩包
3. 解压后运行 `npm install && npm run dev`

### 进阶技巧

#### 材质编辑与替换技巧 ⭐
**1. 参数调整**
- 使用颜色选择器快速选色 (支持HEX/RGB/HSL)
- 拖动滑块时按住Shift可精确调节 (0.01步进)
- 点击数值框可输入精确值 (如 0.75)
- 参数旁有"重置"按钮恢复默认值

**2. 材质预设快速应用**
- 打开材质编辑器 → 选择"预设库"标签页 → 点击预设即可应用

**3. 从预设库替换材质** (推荐)
- 选中对象 → 打开材质编辑器 → "替换"标签页 → 选择预设 → 点击应用 ✅
- 支持多选对象批量替换 (Ctrl+点击多选)
- 替换后可立即预览效果,不满意可Ctrl+Z撤销

**4. 复制粘贴材质**
- 右键源对象 → "复制材质"
- 选中目标对象 (可多选) → 右键 → "粘贴材质" ✅

**5. 批量替换材质**
- 在场景树或视口中多选对象 (Ctrl/Shift + 点击)
- 打开材质编辑器 → 选择新材质 → 点击"应用到所有选中对象" ✅

**6. 按材质槽替换**
- 选中包含多个材质的复杂对象 (如建筑模型)
- 打开材质编辑器 → 查看"材质槽"列表 (如Wall/Glass/Roof)
- 选择要替换的材质槽 → 应用新材质 → 其他材质保持不变 ✅

**7. 使用贴图**
- 点击参数旁的"+"按钮 → 选择贴图文件 → 调整UV和强度

**8. 创建自定义材质**
- 调整参数满意后 → 点击"保存到材质库" → 命名材质 → 以后可直接调用 ✅

**9. 导入外部材质库**
- "材质库"菜单 → "导入" → 选择.dmat文件 → 即可使用他人分享的材质

#### 性能优化技巧

**2. 减少面数**

```typescript
  params: MaterialParams

  if (params.color) material.color.set(params.color);
  if (params.metalness !== undefined) material.metalness = params.metalness;
```

#### 材质编辑与替换 ⭐
```typescript
import { 
  Mesh, 
  MeshStandardMaterial, 
  MeshPhysicalMaterial,
  Material 
} from 'three';

// 材质预设定义
const materialPresets = {
  metal_steel: {
    color: '#71797E',
    metalness: 0.9,
    roughness: 0.3,
  },
  glass_clear: {
    color: '#ffffff',
    metalness: 0.0,
    roughness: 0.0,
    transmission: 0.95,  // 透光率 (MeshPhysicalMaterial)
    opacity: 0.3,
    transparent: true,
  },
  wood_oak: {
    color: '#C4A484',
    metalness: 0.0,
    roughness: 0.8,
  },
  // ... 更多预设
};

// 替换对象材质 (单个)
export function replaceMaterial(
  object: Mesh,
  presetName: string,
  options: {
    replaceAll?: boolean;        // 是否替换所有子对象
    materialSlot?: number;       // 指定材质槽 (默认全部)
    keepMaps?: boolean;          // 保留贴图 (默认否)
  } = {}
) {
  const preset = materialPresets[presetName];
  if (!preset) throw new Error(`材质预设不存在: ${presetName}`);
  
  const { replaceAll = false, materialSlot = -1, keepMaps = false } = options;
  
  // 创建新材质实例 (使用 MeshPhysicalMaterial 支持更多特性)
  const newMaterial = new MeshPhysicalMaterial();
  
  // 应用预设参数
  newMaterial.color.set(preset.color);
  newMaterial.metalness = preset.metalness ?? 0;
  newMaterial.roughness = preset.roughness ?? 1;
  
  if (preset.transmission !== undefined) {
    newMaterial.transmission = preset.transmission;
  }
  if (preset.opacity !== undefined) {
    newMaterial.opacity = preset.opacity;
    newMaterial.transparent = preset.transparent ?? false;
  }
  
  // 如果保留贴图,从旧材质复制
  if (keepMaps && object.material) {
    const oldMaterial = Array.isArray(object.material) 
      ? object.material[materialSlot >= 0 ? materialSlot : 0]
      : object.material;
    
    if (oldMaterial.map) newMaterial.map = oldMaterial.map;
    if (oldMaterial.normalMap) newMaterial.normalMap = oldMaterial.normalMap;
    if (oldMaterial.roughnessMap) newMaterial.roughnessMap = oldMaterial.roughnessMap;
    if (oldMaterial.metalnessMap) newMaterial.metalnessMap = oldMaterial.metalnessMap;
  }
  
  newMaterial.needsUpdate = true;
  
  // 应用到对象
  if (materialSlot >= 0 && Array.isArray(object.material)) {
    // 替换指定材质槽
    object.material[materialSlot] = newMaterial;
  } else if (replaceAll) {
    // 替换所有子对象材质
    object.traverse((child) => {
      if (child instanceof Mesh) {
        child.material = newMaterial;
      }
    });
  } else {
    // 替换对象本身材质
    object.material = newMaterial;
  }
}

// 批量替换材质 (多个对象)
export function batchReplaceMaterials(
  objects: Mesh[],
  presetName: string
) {
  objects.forEach(obj => {
    replaceMaterial(obj, presetName);
  });
}

// 复制材质
export function copyMaterial(sourceObject: Mesh): Material | Material[] {
  return Array.isArray(sourceObject.material) 
    ? sourceObject.material.map(m => m.clone())
    : sourceObject.material.clone();
}

// 粘贴材质到目标对象
export function pasteMaterial(
  targetObject: Mesh, 
  material: Material | Material[]
) {
  targetObject.material = Array.isArray(material) 
    ? material.map(m => m.clone())
    : material.clone();
  targetObject.material.needsUpdate = true;
}

// 创建自定义材质并保存
class MaterialLibrary {
  private materials: Map<string, Material> = new Map();
  
  save(name: string, material: Material) {
    this.materials.set(name, material.clone());
  }
  
  get(name: string): Material | undefined {
    return this.materials.get(name)?.clone();
  }
  
  // 导出材质库 (用于备份/分享)
  export(): string {
    const data = {};
    this.materials.forEach((material, name) => {
      data[name] = this.serializeMaterial(material);
    });
    return JSON.stringify(data);
  }
  
  // 导入材质库
  import(jsonString: string) {
    const data = JSON.parse(jsonString);
    Object.entries(data).forEach(([name, materialData]) => {
      const material = this.deserializeMaterial(materialData);
      this.materials.set(name, material);
    });
  }
  
  private serializeMaterial(material: Material): any {
    // 序列化材质参数为JSON
    return {
      type: material.type,
      color: material.color?.getHexString(),
      metalness: material.metalness,
      roughness: material.roughness,
      // ... 其他参数
    };
  }
  
  private deserializeMaterial(data: any): Material {
    // 从JSON反序列化为材质对象
    const material = new MeshStandardMaterial();
    if (data.color) material.color.set(`#${data.color}`);
    if (data.metalness !== undefined) material.metalness = data.metalness;
    if (data.roughness !== undefined) material.roughness = data.roughness;
    return material;
  }
}
```

```typescript
export async function exportScene(scene: Scene): Promise<Blob> {
  const exporter = new GLTFExporter();
  
  return new Promise((resolve, reject) => {
    exporter.parse(
      scene,
      (result) => {
        const blob = new Blob([result], { type: 'model/gltf-binary' });
        resolve(blob);
      },
      (error) => reject(error),
      { binary: true }
    );
  });
```

## 🐛 常见问题

### Q1: 模型加载失败?
**原因**: 
- 文件格式不支持
- 文件损坏
- 贴图路径错误

**解决**:
- 使用GLB格式 (最稳定)
- 检查文件完整性
- 查看控制台错误信息

### Q2: 编辑卡顿?
**原因**:
- 模型面数过高
- 纹理过大
- 后处理效果过多

**解决**:
- 简化模型或使用LOD
- 压缩纹理 (最大2048)
- 关闭不必要的后处理

### Q3: 导出后无法运行?
**原因**:
- 依赖未安装
- 路径错误
- 浏览器不支持

**解决**:
- 运行 `npm install`
- 检查文件路径
- 使用现代浏览器

### Q4: 保存后刷新丢失?
**原因**:
- 浏览器清除了IndexedDB
- 隐私模式限制

**解决**:
- 及时导出项目备份
- 避免使用隐私模式
